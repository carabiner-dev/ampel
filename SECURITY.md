# Security Policy

## Reporting a Vulnerability

The AMPEL project takes security seriously. If you discover a vulnerability,
please report it responsibly.

### Preferred Method (GitHub Private Vulnerability Reporting)

Please use GitHub’s **Private Vulnerability Reporting** feature to submit your
report:

👉 https://github.com/carabiner-dev/ampel/security/advisories/new

This ensures that maintainers are notified securely and can coordinate a fix
before public disclosure.

### Alternative Contact

If you are unable to use GitHub, or you have any questions, contact us via email:

security@carabiner.dev

## What to Include in a Report

To help us respond quickly and effectively, please include:

- Description of the vulnerability
- Steps to reproduce (proof of concept if possible)
- Impact assessment (what an attacker could do)
- Affected versions or configurations
- Suggested mitigation or fix (if available)

## Response Expectations

- We aim to acknowledge reports within **1 business day**
- We will work with you to validate and remediate the issue
- Once resolved, we will coordinate disclosure and credit (if desired)

## Scope

This security policy applies to:

- The AMPEL core engine
- CLI and APIs
- Policy evaluation logic
- Built-in plugins and integrations
- Supporting repositories (collector, policy compiler, etc)

### Out of Scope

The following are generally out of scope unless they lead to a broader impact:

- Issues in third-party dependencies (should be reported upstream)
- Misconfigurations in user environments
- Hypothetical vulnerabilities without a demonstrable impact

## 🚫 Do Not Disclose Publicly

Please **DO NOT** open public issues for security vulnerabilities.  
Use private reporting channels to avoid putting users at risk.

## Responsible Disclosure

We follow a coordinated disclosure process:

1. Report received and acknowledged
2. Maintainers validate and assess impact
3. Fix is developed and tested
4. Advisory is published (GitHub Security Advisory)
5. Credit is given to the reporter (optional)

## 📄 License

By submitting a vulnerability report, you agree that the information may be
used to improve the project’s security and published as part of a security
advisory.

Thank you for helping keep AMPEL and the open source ecosystem secure!
